8. What Cannot Be Scraped — Explicit Flags
The following items were NOT successfully retrieved via web fetch. These require manual action from the client or a browser session with JavaScript enabled.

8.1 Lazy-Loaded Images (Critical Gap)
The GoDaddy builder uses JavaScript-driven lazy loading. ALL section images below the fold are rendered as 1×1px base64 GIF placeholders in the raw HTML:
    data:image/gif;base64,R0lGODlhAQABAAD/ACwAAAAAAQABAAACADs=
This applies to approximately 12–14 images across Home, About, and Products pages. The actual image URLs are injected at scroll-time via JS — they cannot be retrieved without a headless browser session.
Action Required
Client must either: (a) provide original image files directly, or (b) enable browser automation session so images can be captured via scroll-triggered screenshots.
8.2 Video File — train1.mp4
Video URL identified: //img1.wsimg.com/blobby/go/c941c1b4-2638-46cf-9a59-c33b8e6b9290/video/train1.mp4
This is a GoDaddy blobby storage asset. Direct download may work (no auth required on some GoDaddy blobs) but must be verified. The mp4 was not downloaded during scraping — it requires explicit confirmation before downloading.
Action Required
Client to confirm and provide the video file, or authorize direct download from the blobby URL.
8.3 Contact Form Endpoint
The contact form on the Home page submits to an undisclosed endpoint (GoDaddy's hosted form handler). The form action URL and any server-side logic are not accessible via HTML scraping.
Action Required
New implementation must use a replacement form service (e.g. Formspree, Netlify Forms, or custom API). Client to confirm preferred solution.
8.4 Email Subscribe Endpoint
The footer newsletter subscribe form has an email input and "Sign up" button, but the form submission target and any mailing list integration (e.g. Mailchimp, Klaviyo) are not exposed in the HTML.
Action Required
Client to confirm email platform and provide integration credentials or API key.
8.5 Cookie Consent Behavior
The site shows a "This website uses cookies" banner with Accept/Decline buttons. The cookie categories, consent management platform (CMP), and analytics integrations (e.g. Google Analytics, Hotjar) are not identifiable from the HTML alone.
Action Required
Client to confirm analytics tools in use and desired CMP for the new build.
8.6 Step 3 — "Analyze" Content
In the How It Works section on the Home page, Step 3 (Analyze) is listed as a heading but has no visible body copy in the scraped HTML. This appears to be either collapsed/animated content or a CMS-side gap.
Action Required
Client to provide the intended copy for the Analyze step, or confirm it is intentionally blank.
8.7 reCAPTCHA Site Key
The Google reCAPTCHA integration is confirmed but the site key (data-sitekey attribute) is injected via JS and was not captured in the static scrape.
Action Required
New build requires its own reCAPTCHA site key. Client to register at https://www.google.com/recaptcha/admin.
8.8 Meta Tags, OG Data, and SEO
Page titles are visible from scrape but meta descriptions, Open Graph tags, Twitter Card tags, canonical URLs, and structured data (JSON-LD) were not fully captured in the markdown extraction.
Action Required
SEO audit pass needed. Recommended: provide access to GoDaddy SEO settings or share current meta descriptions.
